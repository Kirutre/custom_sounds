### Pull Requests ðŸ› ï¸
1. **Fork:** the repository and create your branch from `main`.
2. **Set up your environment:** following the steps in the [README.md](README.md).
3. **Branching:** use a descriptive name for tour branch: `git checkout -b feature/destroy-this-extension`.
4. **Keep it clean:** if you change styles, make sure to recompile the Tailwind `output.css`.
5. **Commit your changes:** use descriptive messages (e.g., `feat: add volume slider` or `fix: broken modal on Chrome`).
6. **Push to the branch:** `git push origin feature/destroy-this-extension`.
7. **Open the PR:** describe your changes and link the related _issue_ if there is one.

> [!NOTE]
> Comedy aside, please be as descriptive as possible with the names and descriptions you use.
